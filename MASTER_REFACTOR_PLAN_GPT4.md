# MASTER PLAN REFACTORING (GPT-4 EXECUTION READY)

## Цель

План рассчитан на выполнение более ранними моделями уровня GPT-4:
- задачи атомарные (1 задача = 1 итерация),
- минимальный риск регрессий,
- четкие критерии приемки,
- последовательное приведение проекта к одной архитектуре.

Целевая архитектура:
- единый пайплайн `Vite + ESM`,
- одна точка входа JS,
- один источник истины для каждого UI-блока,
- удаление дублей/legacy,
- усиленная безопасность CSP и базовая доступность.

---

## Правила исполнения для GPT-4

1. Выполнять только одну задачу за раз.
2. Не совмещать архитектурные и визуальные изменения в одной итерации.
3. После каждой задачи выполнять локальную проверку:
   - синтаксис JS,
   - отсутствие новых console errors,
   - ручной smoke по затронутому блоку.
4. Не рефакторить "всё сразу" минифицированные монолиты.
5. Любая удаляемая логика сначала отключается через точку подключения (`index.html`/entry), затем удаляется физически отдельной задачей.

Definition of Done (DoD) для каждой задачи:
- изменены только ожидаемые файлы,
- нет новых ошибок в консоли,
- поведение блока сохранено,
- обновлен краткий changelog в PR/коммите.

## Фаза P2: Безопасность, доступность, производительность

### P2.1 Ужесточить CSP
Проблема: текущая CSP допускает `unsafe-inline` и широкий `https:`.

Действия:
- вынести inline `<script>`/`<style>` во внешние файлы,
- перейти на `nonce`/`sha256` для остаточного inline,
- сузить `script-src/style-src/connect-src` до явных доменов.

Критерии приемки:
- политика CSP строже и не ломает страницу,
- нет inline, требующего `unsafe-inline` для скриптов.

Чекпоинт: [ ] inline script/style вынесены или разрешены через nonce/sha256; [ ] в CSP нет `unsafe-inline` для script-src; [ ] script-src/style-src/connect-src ограничены доменами; [ ] перезагрузка — страница рендерится, в Console нет CSP-ошибок.

---

### P2.2 Исправить доступность reduced-motion + skip-link
Проблема:
- часть тяжелых анимаций игнорирует `prefers-reduced-motion`,
- `.skip-link` скрыт на мобильных.

Действия:
- централизованный guard для анимаций/автоплеев,
- вернуть видимость и корректный фокус-путь `.skip-link` на mobile.

Критерии приемки:
- при `prefers-reduced-motion` интенсивные эффекты отключаются,
- keyboard navigation работает на мобильной ширине.

Чекпоинт: [ ] в коде есть guard для reduced-motion; [ ] при «Reduce motion» молнии/автоплей отключены или упрощены; [ ] .skip-link не скрыт на mobile; [ ] на ширине &lt;900px Tab даёт фокус на skip-link, Enter ведёт к контенту.

---

### P2.3 Сократить CSS debt
Проблема: `assets/css/style.css` очень большой и содержит большое число `!important`.

Действия:

Критерии приемки:

Чекпоинт: [ ] часть стилей в модулях (base/components/layout/utilities); [ ] из index.html убраны выносимые `<style>`; [ ] в приоритетных зонах меньше `!important`; [ ] десктоп и mobile без визуальной регрессии.
