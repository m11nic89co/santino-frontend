# Просмотр сайта на другой машине (live + мониторы)

Репозиторий: **https://github.com/m11nic89co/Santino_WebSite**

---

## 1. На второй машине — клонировать и запустить

```bash
git clone https://github.com/m11nic89co/Santino_WebSite.git
cd Santino_WebSite
npm install
```

**Вариант A — встроенный сервер (без Vite):**
```bash
npm run start
```
Открыть в браузере: **http://localhost:3001**

**Вариант B — после добавления Vite (план P1.1):**
```bash
npm run dev
```
Обычно Vite отдаёт на **http://localhost:5173** (порт смотри в терминале).

## 2. Синхронизация с GitHub

На **машине разработки** (где правите код):
```bash
cd /path/to/Santino_WebSite
git add .
git commit -m "описание изменений"
git push
```

На **второй машине** (где смотрите на мониторах):
```bash
cd Santino_WebSite
git pull
# Перезапустить сервер (Ctrl+C, затем снова npm run start или npm run dev)
```
Либо один раз настроить автопул по таймеру/cron, либо перед показом делать `git pull` вручную.

## 3. Профессиональный вариант: один сервер — все мониторы

- На второй машине один раз поднимаете dev-сервер (`npm run start` или `npm run dev`).
- На этой же машине открываете в браузере нужный URL (localhost:3001 или :5173).
- Разворачиваете окно браузера на нужный монитор или делаете несколько окон на разных мониторах с тем же URL — все показывают один и тот же сайт.

**Если нужно смотреть с телефона/планшета в той же сети:**
- Узнайте IP второй машины: `ipconfig getifaddr en0` (Mac) или `hostname -I` (Linux).
- На телефоне откройте: `http://192.168.x.x:3001` (подставьте свой IP и порт).
- Убедитесь, что фаервол разрешает входящие на порт 3001 (или 5173).

**Если нужен доступ из интернета (без VPN):**
- Временно: [ngrok](https://ngrok.com) — `ngrok http 3001` даёт публичный URL.
- Либо развернуть сборку на GitHub Pages / Netlify / Vercel и открывать уже задеплоенный сайт на мониторах.

---

## 4. Краткий чеклист для второй машины

1. Клонировать репо, `npm install`.
2. Запустить `npm run start` (или `npm run dev` после P1.1).
3. Открыть в браузере указанный URL.
4. При обновлениях: `git pull`, перезапуск сервера при необходимости.
