# Вторая машина (Windows): автоподтяжка после push и два экрана

Репозиторий: **https://github.com/m11nic89co/Santino_WebSite**

Инструкция для Windows: один раз настроил — после каждого `git push` с первой машины на втором экране сайт сам обновляется.

---

## 1. Что нужно установить на Windows

1. **Node.js** — https://nodejs.org (LTS). После установки перезапустите терминал.
2. **Git** — https://git-scm.com/download/win. Установите с опциями по умолчанию.

Проверка в **PowerShell** или **CMD**:
```powershell
node -v
npm -v
git --version
```
Должны вывести версии без ошибок.

---

## 2. Клонировать проект и поставить зависимости

Откройте **PowerShell** (или CMD) и выполните:

```powershell
cd C:\
git clone https://github.com/m11nic89co/Santino_WebSite.git
cd Santino_WebSite
npm install
```

Папку можно выбрать свою, например `D:\Projects\Santino_WebSite` — тогда везде подставляйте свой путь.

---

## 3. Автосинхронизация после push

Идея: на второй машине крутится сервер с автоперезагрузкой страницы и скрипт, который раз в 30 секунд делает `git pull`. После pull файлы меняются → сервер это видит → браузер сам обновляется.

### Шаг A — сервер с автоперезагрузкой

В папке проекта установите и запустите **live-server** (один раз установка, потом только запуск):

```powershell
cd C:\Santino_WebSite
npx --yes live-server --port=3001 --open=/
```

Откроется браузер на **http://127.0.0.1:3001**. Окно можно перетащить на первый или второй монитор.

Чтобы сервер работал в фоне, можно запустить его в отдельном окне PowerShell и не закрывать это окно.

### Шаг B — скрипт автоподтяжки из GitHub

В репозитории лежит скрипт **`scripts/autopull-windows.ps1`**. Он раз в 30 секунд заходит в папку проекта и выполняет `git pull`. Когда приходят новые коммиты, файлы обновляются и live-server перезагружает страницу в браузере.

Запуск (в **втором** окне PowerShell):

```powershell
cd C:\Santino_WebSite
.\scripts\autopull-windows.ps1
```

Если PowerShell ругается на выполнение скриптов, один раз выполните:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Оставьте это окно открытым — скрипт будет крутиться и подтягивать изменения.

---

## 4. Два экрана

- На **первом мониторе**: откройте в браузере **http://127.0.0.1:3001** и разверните окно на весь экран.
- На **втором мониторе**: откройте ещё одну вкладку или окно с тем же адресом **http://127.0.0.1:3001** и разверните на второй экран.

Оба окна показывают один и тот же сайт и обновляются одновременно после `git pull`.

---

## 5. Как вы этим пользуетесь

| Где | Действие |
|-----|----------|
| **Машина 1 (разработка)** | Правите код → `git add .` → `git commit -m "..."` → `git push` |
| **Машина 2 (Windows)** | Ничего не делаете: скрипт сам делает `git pull` раз в 30 сек, live-server сам перезагружает страницу на обоих экранах |

При первом запуске на второй машине нужно один раз:
1. Запустить live-server (Шаг A).
2. Запустить скрипт автопулла (Шаг B).
3. Открыть два окна браузера на двух мониторах.

Дальше достаточно пушить с первой машины — на второй всё обновится само.

---

## 6. Если не хотите live-server

Можно без автоперезагрузки: только автопулл и ручное обновление страницы (F5):

```powershell
npm run start
```
Сайт: **http://localhost:3001**. Скрипт `scripts/autopull-windows.ps1` по-прежнему запускайте — репо будет подтягиваться. После прихода изменений нажимайте F5 в браузере.

---

## 7. Устранение проблем

- **«Скрипт не запускается»** — выполните `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` и снова запустите `.\scripts\autopull-windows.ps1`.
- **Порт 3001 занят** — в команде live-server укажите другой: `npx --yes live-server --port=3020 --open=/`.
- **Страница не обновляется** — убедитесь, что запущен и live-server, и скрипт autopull; подождите до 30 секунд после push.
