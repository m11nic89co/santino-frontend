# Release Runbook — SANTINO v2 Rewrite

Инструкция по сборке, проверке и деплою (Phase 11). Работа ведётся только в папке `/rewrite/`.

---

## 1. Сборка production

Из корня **rewrite**:

```bash
cd /path/to/project/rewrite
npm ci
npm run build
```

- `npm ci` — ставит зависимости по `package-lock.json` (рекомендуется для CI/релизов).
- `npm run build` — запускает `tsc -b` и `vite build`. Результат в `rewrite/dist/`.

Артефакты:
- `dist/index.html` — точка входа.
- `dist/assets/*.js` — чанки (в т.ч. `vendor-gsap-*.js`, `vendor-swiper-*.js`, основной бандл приложения).
- `dist/assets/*.css` — сведённые стили.

---

## 2. Проверки перед деплоем

Выполнять до выката:

| Шаг | Команда | Назначение |
|-----|--------|------------|
| 1 | `npm run typecheck` | Проверка типов TypeScript |
| 2 | `npm run lint` | ESLint без предупреждений |
| 3 | `npm run build` | Успешная production-сборка |
| 4 | `npm run preview` | Локальный просмотр `dist/` (ручная проверка в браузере) |

Опционально (при настроенных тестах):

- `npm run test` — юнит-тесты.
- `npm run test:e2e` — E2E (Playwright).

Рекомендуется не деплоить при падении typecheck, lint или build.

---

## 2.1 Go/No-Go Metrics

Перед принятием решения о деплое проверить (вручную или через мониторинг):

| Метрика | Порог | Описание |
|--------|--------|----------|
| TTI (Time to Interactive) | &lt; 2,5 с | Страница интерактивна (навигация, Hero, карусель доступны). |
| CLS (Cumulative Layout Shift) | &lt; 0,1 | Нет заметных сдвигов контента при загрузке. |
| Консоль браузера | 0 ошибок | В логе продакшн-страницы не должно быть необработанных ошибок. |

**No-Go:** при нарушении любого из порогов — отложить деплой до выяснения причины.

---

## 2.2 Rollback Triggers (когда откатываемся)

Выполнить **немедленный откат** на предыдущую сборку, если:

- Падают **P0-контрактные тесты** (E2E/ручные критические сценарии).
- **Hero-блок визуально сломан** (нет заголовка/CTA, анимация не запускается или ломает вёрстку).
- На проде зафиксированы **критические ошибки в консоли** или падения JS до интерактивности.

После отката — зафиксировать инцидент и провести разбор.

---

## 3. Локальный просмотр собранного билда

После `npm run build`:

```bash
npm run preview
```

Открыть указанный URL (обычно `http://localhost:4173`). Проверить: загрузка, навигация, Hero, карусель Collection, тикер, reduced-motion (если включён в ОС).

---

## 3.1 Post-deploy Smoke Test (чек-лист)

После выката на целевое окружение пройти пункты:

- [ ] **Карусель крутится** — в секции «Коллекция» слайды переключаются (свайп/кнопки/пагинация), эффект coverflow отображается.
- [ ] **Меню открывается** — на мобильной ширине кнопка «гамбургер» открывает/закрывает меню, навигация по секциям работает.
- [ ] **Анимации есть** — Hero: заголовок и CTA появляются с анимацией (при отключённом reduced-motion); тикер движется в зоне видимости.
- [ ] **Нет ошибок в консоли** — во время сценария выше в DevTools Console нет красных ошибок.

При провале любого пункта — считать smoke failed и действовать по Rollback Triggers.

---

## 4. Откат (rollback)

- Деплой — через вашу инфраструктуру (обратная выкатка предыдущей версии артефактов из `dist/`).
- При проблемах после деплоя: вернуть предыдущий билд и при необходимости откатить конфиг/трафик по процедурам вашей платформы.

В кодовой базе rewrite не хранятся данные пользователей; откат сводится к возврату предыдущей сборки.

---

## 5. Зависимости и чанки

- **GSAP** и **Swiper** подключаются только в `src/main.ts` и передаются в адаптеры; в фичах и адаптерах прямого импорта вендоров нет.
- Vite разбивает сборку: тяжёлые библиотеки вынесены в отдельные чанки (`manualChunks`: `vendor-gsap`, `vendor-swiper`), чтобы кэш вендоров не инвалидировался при изменении кода приложения.

Версия: Phase 11 + hardened (Go/No-Go, Rollback Triggers, Post-deploy Smoke).
