# Журнал архитектурных решений (ADR)

## Назначение

Журнал фиксирует архитектурные решения, принятые в рамках программы Rewrite. Каждая запись должна содержать: контекст, решение, последствия и статус.

## Формат записи ADR

- **ADR-XXX** — уникальный номер.
- **Дата** — дата фиксации.
- **Статус** — Proposed | Accepted | Deprecated | Superseded.
- **Контекст** — что требовалось решить.
- **Решение** — принятый вариант.
- **Последствия** — плюсы, минусы, ограничения.

## Записи

### ADR-001 — Физическая изоляция rewrite

- **Дата:** Phase 0  
- **Статус:** Accepted  
- **Контекст:** Исключить смешивание legacy и новой кодовой базы.  
- **Решение:** Вся новая разработка ведётся только в `/rewrite/` и `/docs/rewrite/`. Legacy остаётся в текущем дереве в режиме read-only.  
- **Последствия:** Чёткая граница; невозможность случайного изменения legacy; необходимость явного cutover на этапе Phase 12.

*Дальнейшие ADR добавляются по мере принятия решений в ходе фаз.*
