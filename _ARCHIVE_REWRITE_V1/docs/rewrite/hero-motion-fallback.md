# Hero Motion — Fallback при отключённом motion (Phase 7)

## Когда motion отключён

1. **Система:** у пользователя включено «Уменьшить движение» (prefers-reduced-motion: reduce).
2. **Результат:** на `<html>` добавляется класс `reduced-motion`; Hero не проигрывает анимацию появления (duration передаётся как 0 через IAnimationPort).

## Поведение

- Контент Hero (заголовок, CTA) остаётся видимым и доступным.
- CTA ведёт на #contact; клик и клавиатура работают как обычно.
- Визуально элементы отображаются сразу, без анимации (без деградации функциональности).

## Проверка

- В DevTools включить «Emulate CSS media feature prefers-reduced-motion: reduce».
- Убедиться: у `<html>` есть класс `reduced-motion`, блок hero не анимируется при переключении на секцию 0.
